Provides the basic tools to create, use or delete the Astral engine. At first glance, this simple bundle has only a large set of interface and some very basic classes. The implementations are provided inside the core package. When developing an astral application, you do not need to requires the core.

Access to the engine

The astral engine has been built under OSGi-iPOJO. When the api and the core bundles are deployed inside an OSGi container (like felix). You can access to the engine via a requirement of services from iPOJO. 
    public class Foo {
        @Requires
        private AstralEngine astral;
    }
Then the framework will find the only implementation that matches this interface. Now you can access to all the methods provided by the engine, notably the core creation.

Create a core

A core in the sense of astral is a complete toolset to create, modify, run and delete a single query. Each core is unique and can accept only one query. The engine provide a direct method to create a new core. Starting from this part, the user can build a new query by hand with the {@link fr.lig.sigma.astral.operators.OperatorFactory} et all source managers available. An engine can contain multiple cores indeed.

Run a query

Once the {@link fr.lig.sigma.astral.common.structure.Entity} is built, the user must register it inside the engine. When it's done, the user has a {@link fr.lig.sigma.astral.query.QueryRuntime} to control the cycle of life of the query. By doing a start, the engine launch an execution thread.

What the engine does NOT do

The engine itself does not provide :
  • A query builder that transforms a string to an {@link fr.lig.sigma.astral.common.structure.Entity}
  • A distributed execution of the query
  • Any GUI
  • Any real source mapping. As each source has their specific behaviour, only very basic sources are implemented as demonstration.
Fortunately, other bundles of the AStrAL distribution does provide those functionality.
