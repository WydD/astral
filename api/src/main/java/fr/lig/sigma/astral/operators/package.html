Base of all operators implementation considering the definitions of AStrAL. As a reminder, the operator definitions can be found here. This package gives the minimum framework to write quickly any type of operators. With the abstract classes like {@link fr.lig.sigma.astral.operators.RelationOperator} or {@link fr.lig.sigma.astral.operators.StreamOperator}, implementing an operator can be nearly reduced to it's core algorithm.

Common rules to define an operator

Each operator, are classes that implements the interface {@link fr.lig.sigma.astral.operators.Operator} as well as their right output entity type. An interface must be defined for that, like {@link fr.lig.sigma.astral.operators.pi.Pi} for example with an unique function that will initialize the operator with its input and parameters. The implementation must follow very simple rules to define a good operators.
  • Extends on of the two abstract operators that extends {@link fr.lig.sigma.astral.common.structure.EntityContainer}
  • Do a setOutput as soon as enough details about the attributes and names are gathered
  • Then only, do a addInput to indicate to the scheduler the dependencies of the operator, and if it is the case, to be registered as an input listener.
  • Implement your algorithm inside the processEvent of {@link fr.lig.sigma.astral.common.event.EventProcessor}
If your operator is a composite and needs to create other operators, just extends the interface with {@link fr.lig.sigma.astral.operators.OperatorFactory}. The core will detect it at the instanciation and will attach the right one to this component.
